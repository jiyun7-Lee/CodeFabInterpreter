## Chapter 1: 프로젝트 개요

CodeFab = 인터프리터. Custom Language를 받아 공장처럼 처리.

파이프라인: Assembler Unit → Checker Unit → Executor Unit
- Assembler: 소스코드 → Token List → AST(Expr/Stmt 트리)
- Checker:   AST 의미 오류 검사 (DFS 순회)
- Executor:  AST 실행 (DFS 순회)

금지 예시(Page 8): #define 매크로로 언어 키워드를 우회하는 방식은 안 됨.

## Chapter 2: Assembler Unit

### 2-1. Tokenizer

Token 종류: 키워드(if/for/var 등), 식별자, 리터럴(숫자/문자열), 연산자, 구분자

기본 TokenType 표:
| TokenType             | 예시    | 비고       |
|-----------------------|---------|------------|
| LEFT_PAREN/RIGHT_PAREN| ( )     | 그룹핑     |
| LEFT_BRACE/RIGHT_BRACE| { }     | 블록 스코프 |
| SEMICOLON             | ;       | 구분자     |
| PLUS/MINUS/STAR/SLASH | + - * / | 산술연산   |
| EQUAL                 | =       | 할당       |
| GREATER/LESS          | > <     | 비교       |
| IDENTIFIER            | x, y    | 식별자     |
| STRING                | "hello" | 문자열     |
| NUMBER                | 3, 3.14 | 숫자(double)|
| VAR                   | var     | 변수 선언  |
| IF/ELSE               | if else | 조건문     |
| FOR                   | for     | 반복문     |
| TRUE/FALSE            | true false| 불리언   |
| AND/OR                | and or  | 논리       |
| PRINT                 | print   | 출력문     |
| EOF                   |         | 스트림 끝  |

※ 필요 시 추가 가능

Token 클래스: type, origin(원본 문자열), [필요시 literal 값 추가]

### 2-2. Expression (Expr)

값을 평가(evaluate)하는 노드. 실행하면 값 하나로 치환됨.
Expr 내부에 Stmt를 Child로 갖는 것은 금지.

| Expr 종류  | 예시              | 설명            |
|------------|-------------------|-----------------|
| Literal    | 3, "hi", true     | 값 생성         |
| Variable   | a                 | 변수 참조       |
| Assign     | a = 3             | 변수 대입       |
| Binary     | 1+2, a>0, 2*3     | 이항 연산       |
| Unary      | -x, !flag         | 단항 연산       |
| Grouping   | (1 + 2)           | 괄호            |
| Logical    | a and b, a or b   | 논리 연산       |

연산자 우선순위는 트리 구조에 반영됨:
  a + b * 3  →  BinaryExpr(+)
                 ├── Variable(a)
                 └── BinaryExpr(*)
                      ├── Variable(b)
                      └── Literal(3)

### 2-3. Statement (Stmt)

값 반환 없이 동작을 수행하는 노드. 트리 루트는 항상 Stmt.
Token은 노드가 아닌 각 노드의 필드로 보관.

| Stmt 종류   | 예시                        |
|-------------|----------------------------|
| Expression  | a + 1; (Expr Wrapper)       |
| Print       | print a;                    |
| VarDeclare  | var a = 3;                  |
| Block       | { … }                       |
| If          | if (a > 0) { … } else { … } |
| For         | for(var i=0; i<3; i=i+1){ … }|

### 2-4. 가공 작업 (Parser)

문법 규칙: if Stmt → "if" "(" expression ")" statement ("else" statement)?
파싱 과정: 토큰 하나씩 소비 → 노드 구성 → 트리로 조립
실행 불필요한 토큰(=, ; 등)은 노드로 만들지 않음.
문법 오류(예: var a = 3 +;) → 파서 단계에서 오류 발생.

구현 지침:
- 세부 문법 규칙과 각 노드의 필드는 팀에서 결정
- 제공되는 테스트용 스크립트를 전부 통과해야 함
- 스크립트 문법은 custom 언어에 맞게 변경 가능

## Chapter 3: Checker Unit

역할: AST를 DFS 순회하며 의미 오류를 검출 (실행 전).

검사 항목:
1. 변수 중복 선언 (같은 블록 내)
   { var a = "first"; var a = "second"; }  → Error

2. 자기 초기화 참조
   { var a = a + 1; }  → "[2번째 줄] 자신의 초기화식에서 지역변수를 읽을 수 없습니다."

---

## Chapter 4: Executor Unit

역할: AST를 DFS 순회하며 실제 실행.

### 변수 저장소 (Environment/Scope)
- 블록 { } 진입 시 새 Local 저장소 생성, 종료 시 소멸
- 변수 탐색: 현재 저장소 → 상위 → … → Global

### Runtime 오류 3종
1. 타입 불일치:  3 - "hello" → "[1번째줄] 피연산자는 반드시 숫자여야 한다."
2. 미정의 변수:  x = 5 (선언 없음) → "[1번째줄] 미정의된 변수 'x'"
3. 0으로 나누기: a = 3 / 0 → "[1번째줄] 0으로 나눈 오류"
